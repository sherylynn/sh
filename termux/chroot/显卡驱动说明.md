## 显卡驱动说明

### 概述

本系统支持两种显卡驱动模式，系统会根据硬件配置自动选择合适的驱动，同时也支持通过配置文件强制指定驱动模式。

---

### 驱动模式对比

| 特性 | 原生驱动 (freedreno/kgsl) | 虚拟驱动 (virgl) |
|------|---------------------------|------------------|
| **适用CPU** | Oryon (至尊版) | 其他ARM CPU |
| **性能** | 高，直接调用硬件 | 中等，经过虚拟化层 |
| **兼容性** | 部分应用可能不兼容 | 兼容性较好 |
| **资源占用** | 低 | 较高 |
| **3D加速** | 原生支持 | 虚拟化支持 |

---

### 自动选择机制

系统启动时会自动检测硬件配置，优先级如下：

1. **优先检测强制配置文件**：
   - 如果存在 `/sdcard/Download/使用虚拟显卡.txt` 文件
   - 无论CPU类型，强制使用 virgl 虚拟驱动

2. **检测CPU类型**：
   - Oryon CPU：使用原生 freedreno/kgsl 驱动
   - 其他CPU：使用 virgl 虚拟驱动

---

### 强制使用虚拟显卡

如需强制使用虚拟显卡（例如解决兼容性问题），只需创建一个空文件：

```bash
touch /sdcard/Download/使用虚拟显卡.txt
```

删除此文件即可恢复自动选择：

```bash
rm /sdcard/Download/使用虚拟显卡.txt
```

---

### 涉及的脚本文件

以下脚本会根据驱动模式做出不同处理：

| 脚本文件 | 原生驱动行为 | 虚拟驱动行为 |
|----------|-------------|-------------|
| `win-git/server_noVNC.sh` | 设置 `MESA_LOADER_DRIVER_OVERRIDE=kgsl` | 设置 `GALLIUM_DRIVER=virpipe` |
| `win-git/mesa.sh` | 安装 Oryon 专属驱动 | 跳过驱动安装 |
| `termux/termux_service_virgl.sh` | 删除 virgl 服务 | 创建 virgl 服务 |
| `termux/server_virgl.sh` | 不启动 `virgl_test_server_android` | 启动 `virgl_test_server_android` |
| `termux/chroot/termux_all_in_one.sh` | 不启动/停止 virgl 服务 | 启动/停止 virgl 服务 |
| `termux/chroot/noVNC.sh` | 设置原生驱动环境变量 | 设置 virgl 环境变量 |
| `termux/chroot/x11.sh` | 设置原生驱动环境变量 | 设置 virgl 环境变量 |
| `termux/chroot/virgl.sh` | 不启动 `virgl_test_server_android` | 启动 `virgl_test_server_android` |

### 技术细节

#### 原生驱动 (freedreno/kgsl)

适用于搭载 Oryon CPU 的设备：
- 使用高通 Adreno GPU 的原生驱动
- 环境变量：`MESA_LOADER_DRIVER_OVERRIDE=kgsl`
- 需要安装专门的 turnip 驱动包
- 无需运行额外服务

#### 虚拟驱动 (virgl)

适用于其他 ARM 设备或强制使用虚拟显卡时：
- 通过 virglrenderer 实现 GPU 虚拟化
- 环境变量：`GALLIUM_DRIVER=virpipe`
- 需要运行 `virgl_test_server_android` 服务
- 通过 `sv` 管理 virgl 服务

---

### 服务管理

virgl 服务是虚拟驱动模式的核心组件：

**启动 virgl 服务：**
```bash
sv up virgl
```

**停止 virgl 服务：**
```bash
sv down virgl
```

**查看服务状态：**
```bash
sv status virgl
```

### 启动流程

#### 启动链路总览

系统支持两种启动方式，两种方式都会根据配置选择显卡驱动：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    termux_all_in_one.sh start                       │
└───────────────────────────┬─────────────────────────────────────────┘
                            │
            ┌───────────────┼───────────────┐
            ▼               ▼               ▼
    ┌─────────────┐  ┌─────────────┐  ┌─────────────┐
    │ sv up virgl │  │sv up pulseaudio││  sv up x11  │
    └──────┬──────┘  └─────────────┘  └──────┬──────┘
           │                                  │
           ▼                                  ▼
    ┌─────────────┐                  ┌─────────────┐
    │server_virgl.sh│              │server_x11.sh │
    └──────┬──────┘                  └─────────────┘
           │
           ▼
    virgl_test_server_android
            │
            └────────────────────────┐
                                     │
                                     ▼
                         start_chroot_container()
                                     │
                                     ▼
                          server_noVNC.sh (chroot内)
                                     │
                                     ▼
                          设置显卡驱动环境变量
```

#### 方式一：使用 termux-services（推荐）

这是 `termux_all_in_one.sh` 使用的方式：

```
1. termux_all_in_one.sh start
    └─> 检测 /sdcard/Download/使用虚拟显卡.txt
        ├─> 存在 -> sv up virgl
        └─> 不存在 -> 检测 CPU 类型
            ├─> Oryon -> 跳过 virgl 服务
            └─> 其他 -> sv up virgl

2. sv up virgl
    └─> $PREFIX/var/service/virgl/run
            └─> server_virgl.sh
                    └─> 检测 /sdcard/Download/使用虚拟显卡.txt
                        ├─> 存在 -> 启动 virgl_test_server_android
                        └─> 不存在 -> 检测 CPU 类型
                            ├─> Oryon -> 不启动
                            └─> 其他 -> 启动 virgl_test_server_android

3. chroot 容器启动后
    └─> server_noVNC.sh
            └─> 检测 /sdcard/Download/使用虚拟显卡.txt
                ├─> 存在 -> 设置 GALLIUM_DRIVER=virpipe
                └─> 不存在 -> 检测 CPU 类型
                    ├─> Oryon -> 设置 MESA_LOADER_DRIVER_OVERRIDE=kgsl
                    └─> 其他 -> 设置 GALLIUM_DRIVER=virpipe
```

#### 方式二：直接运行脚本

这是手动启动时使用的方式：

```
直接运行 chroot/noVNC.sh 或 chroot/x11.sh
    └─> 检测 /sdcard/Download/使用虚拟显卡.txt
        ├─> 存在 -> 启动 virgl_test_server_android + 设置 virgl 环境变量
        └─> 不存在 -> 检测 CPU 类型
            ├─> Oryon -> 不启动 virgl + 设置原生驱动环境变量
            └─> 其他 -> 启动 virgl_test_server_android + 设置 virgl 环境变量
```

#### 脚本调用关系

| 启动入口 | 服务管理 | 实际执行脚本 | 驱动检测位置 |
|----------|----------|--------------|--------------|
| `termux_all_in_one.sh` | termux-services | `server_virgl.sh` | `server_virgl.sh` + `server_noVNC.sh` |
| `chroot/noVNC.sh` | 直接运行 | 自身 | 自身内部 |
| `chroot/x11.sh` | 直接运行 | 自身 | 自身内部 |
| `chroot/virgl.sh` | 直接运行 | 自身 | 自身内部 |

#### 驱动检测优先级

系统在多个环节都会进行驱动检测，确保一致性：

1. **服务启动阶段**：决定是否启动 virgl 服务
2. **服务执行阶段**：决定是否启动 virgl_test_server_android
3. **chroot 进入阶段**：决定设置哪些环境变量

所有环节使用相同的检测逻辑，确保驱动模式一致。

### 常见问题

**Q: 如何确认当前使用的驱动模式？**

A: 检查环境变量或查看启动日志：
```bash
echo $GALLIUM_DRIVER
echo $MESA_LOADER_DRIVER_OVERRIDE
```

**Q: 使用原生驱动时部分应用崩溃怎么办？**

A: 创建强制使用虚拟显卡文件来切换到 virgl 模式：
```bash
touch /sdcard/Download/使用虚拟显卡.txt
```

**Q: 虚拟驱动性能不足怎么办？**

A: 如果设备是 Oryon CPU，确保没有强制虚拟显卡文件，系统会自动使用原生驱动。

**Q: 如何查看 virgl 服务状态？**

A: 使用 sv 命令查看：
```bash
sv status virgl
```

**Q: 修改驱动模式后需要重启吗？**

A: 需要重启相关服务或重新进入 chroot 环境才能生效。

### 故障排除

#### 虚拟显卡模式无法启动

1. 检查 virgl 服务状态：
   ```bash
   sv status virgl
   ```

2. 检查 virgl_test_server_android 是否运行：
   ```bash
   pgrep virgl_test_server_android
   ```

3. 查看服务日志：
   ```bash
   cat $PREFIX/var/service/virgl/log/current
   ```

#### 原生显卡模式显示异常

1. 检查是否存在强制虚拟显卡文件：
   ```bash
   ls -la /sdcard/Download/使用虚拟显卡.txt
   ```

2. 检查是否安装了正确的驱动：
   ```bash
   dpkg -l | grep mesa
   ```

3. 查看启动日志确认驱动加载情况

---

### 总结

| 场景 | 推荐驱动 | 说明 |
|------|---------|------|
| Oryon CPU 设备 | 原生驱动 | 性能最佳 |
| 非 Oryon CPU 设备 | 虚拟驱动 | 默认选择 |
| 应用兼容性问题 | 虚拟驱动 | 创建强制文件切换 |
| 调试/测试 | 虚拟驱动 | 创建强制文件切换 |
